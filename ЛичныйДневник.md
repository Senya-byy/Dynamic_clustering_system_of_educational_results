## Деплой и окружения

### Render (production)
- Backend поднят как **Web Service** (Docker, `python:3.11-slim`) из-за проблемы сборки зависимостей на Python 3.14 (падала установка Pillow).
- Frontend поднят как **Static Site**; для сборки указано `npm ci && npm run build`, publish directory `dist`.
- Исправлена ошибка путей вида `backend/backend` (нельзя “удваивать” Root Directory и Dockerfile Path).
- Исправлена проблема **404 при обновлении страниц** в SPA (Vue Router history): на Render настроено **Redirects/Rewrites** `/* → /index.html` (Rewrite).

### База данных
- Для production используется **Render PostgreSQL** (`classqr-db`).
- Переключение сделано через `DATABASE_URL`.
- Проверка корректности подключения: `GET /health` возвращает `db: ok`.

### Bootstrap администратора (seed выключен)
Проблема:
- в production `SEED_DATA=false`, поэтому дефолтных пользователей нет → логин даёт `401 Invalid credentials`.

Решение:
- добавлен одноразовый эндпоинт `POST /api/admin/bootstrap`:
  - работает только если задан `BOOTSTRAP_TOKEN` в env
  - требует заголовок `X-Bootstrap-Token`
  - создаёт администратора только если админа ещё нет
- после инициализации админа `BOOTSTRAP_TOKEN` нужно удалить (endpoint станет 404).
