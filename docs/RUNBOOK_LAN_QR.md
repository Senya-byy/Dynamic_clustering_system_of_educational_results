# RUNBOOK_LAN_QR — запуск и QR с телефона

Этот runbook предназначен для преподавателя/проверяющего: как запустить систему и сделать так, чтобы QR открывался на телефоне по Wi‑Fi.

## 1) Предусловия
- Телефон и ноутбук **в одной Wi‑Fi сети** (не LTE, не гостевая сеть с изоляцией клиентов).
- Порты:
  - Backend: **5001**
  - Frontend: **5173**

## 2) Как узнать IP ноутбука (LAN)

### macOS
```bash
ipconfig getifaddr en0
```
Если пусто, попробуйте:
```bash
ipconfig getifaddr en1
```

### Linux
```bash
hostname -I
```
или
```bash
ip a
```

### Windows (PowerShell)
```powershell
ipconfig
```
Ищите `IPv4 Address` у активного Wi‑Fi адаптера.

## 3) Проверка доступности портов

### Проверка с ноутбука
```bash
curl -I http://127.0.0.1:5173
curl -I http://127.0.0.1:5001
```

### Проверка с телефона
Откройте в браузере телефона:
- `http://<ВАШ_IP>:5173`

Если страница не открывается:
- скорее всего Vite слушает только `127.0.0.1` (см. раздел «Типовые проблемы»),
- или сеть изолирует клиентов,
- или firewall блокирует порт.

## 4) Запуск (локально, рекомендовано для разработки)

### macOS/Linux
```bash
./scripts/dev.sh
```
Скрипт:
- проверит порты,
- запустит backend и frontend,
- выведет LAN URL вида `http://<IP>:5173` для QR.

### Windows
Запустите:
```bat
scripts\dev.bat
```

## 5) Запуск через Docker

1) Создайте `.env` по примеру:
```bash
cp .env.example .env
```
2) Укажите `QR_FRONTEND_ORIGIN` (IP ноутбука!):
```env
QR_FRONTEND_ORIGIN=http://192.168.1.100:5173
```
3) Запуск:
```bash
docker compose up --build
```

## 6) Как получить QR и проверить
1) Откройте преподавательскую панель на **LAN URL**: `http://<IP>:5173`
2) Войдите под преподавателем.
3) Перейдите в **«Пары и QR»** → создайте сессию → включите «Живой QR».
4) Сканируйте QR телефоном → должно открыться `/join` на **том же IP**.

## 7) Типовые ошибки и что делать

### «nonce expired» / «Неверный или просроченный QR»
Причина: QR обновляется, `nonce` живёт ~3.5 сек.
- Сканируйте **актуальный** QR.
- Убедитесь, что часы телефона не сильно «уехали».

### «device already bound» / «устройство привязано к другому аккаунту»
Причина: устройство уже закреплено за другим логином в рамках сессии.
- Выйдите и войдите под тем аккаунтом, которым сканировали QR.
- Или очистите site data/кэш (если `device_id` хранится в localStorage).

### «session closed» / «Сессия не активна»
Причина: преподаватель закрыл сессию.
- Создайте новую сессию.

### Телефон открывает «не то» или ничего не открывает
Проверки:
- QR не должен содержать `localhost` (телефон открывает localhost **на себе**).
- Откройте `http://<IP>:5173` вручную на телефоне.
- Убедитесь, что Vite слушает `0.0.0.0` (в логе Vite должна быть строка `Network: http://<IP>:5173/`).

### Firewall блокирует доступ
- macOS: System Settings → Network → Firewall → разрешить входящие для node/python (или выключить для теста).
- Windows: Windows Defender Firewall → разрешить входящие для порта 5173/5001.
