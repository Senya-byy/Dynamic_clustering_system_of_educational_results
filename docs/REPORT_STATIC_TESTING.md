# REPORT_STATIC_TESTING — отчёт (статическое тестирование)

Дата: 2026-04-27  
Проект: ClassQR  
Окружение: ревью репозитория (без запуска)

## 1) Что проверялось
- структура репозитория, наличие “мусорных” файлов
- конфиги деплоя/запуска (Render, Docker, env)
- архитектура и документация в `docs/`
- базовые риски безопасности (JWT/CORS/RBAC) по коду

## 2) Итоги (кратко)
- **SPA роутинг (Vue Router history)**: на static hosting требует rewrite на `index.html` (иначе 404 при F5).
- **RBAC/JWT**: есть декораторы `token_required` и `role_required`, доступ к teacher/admin/student эндпоинтам ограничен.
- **CORS**: в production по умолчанию безопасно (пустой allowlist), origins задаются через `CORS_ORIGINS`.
- **Документация**: добавлен гайд по Render и документ по тестированию.
- **Репозиторий**: удалены локальные артефакты IDE/venv/cache и дубликаты документов.

## 3) Найденные проблемы/риски
### 3.1 Исторический риск: SPA 404 при обновлении
- **Причина**: `createWebHistory()` на фронте + static hosting без rewrite.
- **Фикс**: для Render Static Site настроить `/* → /index.html` (Rewrite).

### 3.2 Технический долг: 2 разных JWT-утилиты
- `backend/services/auth_service.py` формирует JWT с ключом `id`, что соответствует `backend/middleware/auth_middleware.py`.
- Дополнительно существует `backend/utils/jwt_utils.py` (другая форма payload `user_id`), который сейчас не является источником истины.
- **Риск**: при дальнейшем расширении можно случайно начать декодировать “не тот” формат.

## 4) Что изменено по итогам статического тестирования
- В `docs/` добавлены/актуализированы документы по Render и тестированию.
- Репозиторий очищен от локальных артефактов (IDE/venv/cache) и дублей.
